    General

    Introduction

    Roadtripper API is a REST API, each request use the HTTP protocol.
    Each request paths (exept /ping) must be prefixed by the current major version of the api server. Since we are in version 1.-.-, paths must be prefixed by /v1
    For example, the url of the login request ( /auth/login ) is /v1/auth/login

    Security

    The API handle two type of authorization. To know which one you have to implement for a request, look at the request TAGs (top-right coner of the request description).

    Client Auth

    Request need to be authenticated by HTTP Basic Authentification. Add your client secret to the HTTP header of your request:


    
Authorization : Basic YOUR_CLIENT_SECRET

    User Auth

    Request need to bo authenticated by Access token. Add your accessToken to the HTTP header of your request: 


    
Authorization : Bearer YOUR_ACCESS_TOKEN

    To obtain an accessToken, use the /auth/login route.

    Auth

    Requests related to authorization

    GET /auth/login
    Client Auth

    Login the user and reply a accessToken

    This operation register the user if it is the first time he use the application.

    Parameters

    Name Description Required In Schema
    facebookToken The facebook access token true body string

    Responses

    Http Status Code Description Schema
    200 Operation succeed Access token reply

    Errors

    HTTP status Code SubCode Message Comment
    400 1 Invalid request, (parameters missing)

    Check request parameters twice
    401 1 1 Request need to be authorized by Client Auth (check documentation for more details)

    See Client Auth paragraph

    401 3 1 Invalid facebook access token, please update facebook token (POST /auth/facebooktoken)

    See POST /auth/facebooktoken

    GET /auth/token
    User Auth

    Get information about the current token (included in the request header)

    Responses

    Http Status Code Description Schema
    200 Valid token Token information reply

    Errors

    HTTP status Code SubCode Message Comment
    401 1 2 Request need to be authorized by User Auth (check documentation for more details)

    See User Auth paragraph

    401 2 1 Token expired, please use POST /auth/renew to obtain a new token

    See POST /auth/renew

    401 2 2 Invalid token, please use login again to get a new token

    See User Auth paragraph
